# 项目管理

## 目标

确保项目数据可读取、可追踪、可恢复。

## 数据建议

- 项目文件统一存储在 AppData 的 `ClipFlow/*.json`
- 读写优先走 Tauri Rust 命令，前端 FS 仅兜底
- 保持字段兼容：`videoPath` / `videoUrl` / `videos[]`

## 页面流程建议

1. 项目列表：只展示可解析数据
2. 项目编辑：手动保存优先，自动保存可开关
3. 项目详情：读取后统一归一化
4. 删除项目：调用统一删除接口并刷新列表

## 稳定性建议

- 避免同一能力出现两套服务实现
- 自动保存仅对核心内容触发，不因输入框抖动触发
- 开发环境避免 StrictMode 双执行带来的误判

## 发布前检查

- 项目新建/编辑/删除链路可用
- 首页最近项目可正确显示
- 编辑页无循环跳转
- 打包可产出 `.dmg`
