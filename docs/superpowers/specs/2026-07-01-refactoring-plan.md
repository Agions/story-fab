# StoryFab 架构深度分析与系统性重构计划

## 📊 项目概况

**项目规模**：
- 前端：49,495 行 TypeScript（406 个文件）
- 后端：7,739 行 Rust（94 个文件）
- 总计：~57,000 行代码

**技术栈**：
- 前端：React 18 + TypeScript 5 + Vite 6 + TailwindCSS 4 + Zustand
- 后端：Tauri 2 + Rust 1.77+
- 架构：多层服务架构 + Pipeline 模式 + Tauri IPC

**主要模块**：
```
src/
├── core/               # 核心业务逻辑
│   ├── pipeline/       # Pipeline 引擎和步骤
│   ├── services/       # 服务层（AI/导出/字幕/视频/commentary）
│   ├── config/         # 配置管理
│   ├── tauri/          # Tauri IPC 方法
│   ├── errors/         # 错误处理
│   └── utils/          # 工具函数
├── components/         # React 组件
├── context/            # React Context
├── hooks/              # 自定义 Hooks
└── stores/             # Zustand Stores
```

## 🎯 重构目标

1. **架构升级**：优化模块耦合，提升可扩展性
2. **全局规范化**：统一命名，消除死代码
3. **消除重复**：DRY 原则，抽象公共组件
4. **极致精简**：删除冗余逻辑，优化代码密度

## 🚀 团队分工

**Agent Team**（5个并行分析agent）：

1. **架构分析agent**：分析模块依赖、设计模式、架构问题
2. **代码质量agent**：识别死代码、重复代码、命名问题
3. **性能分析agent**：识别性能瓶颈、优化机会
4. **TypeScript专项agent**：类型系统优化、类型安全
5. **Rust专项agent**：后端代码优化、性能提升

每个agent将输出：
- 发现的问题列表（按严重程度排序）
- 具体的重构建议
- 预估的影响范围
