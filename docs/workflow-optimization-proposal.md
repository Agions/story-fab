# StoryFab 工作流优化建议

> 基于短剧/影视解说行业最佳实践 vs 当前项目能力差距分析
> 日期：2026-07-07

---

## 1. 行业工作流基准 (Industry Benchmark)

短剧/影视解说创作的标准工作流包含 **8 个环节**：

```
┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐
│ ① 选题  │ → │ ② 素材  │ → │ ③ 拆条  │ → │ ④ 文案  │
│ 策划     │   │ 准备     │   │ 切片     │   │ 生成     │
└─────────┘   └─────────┘   └─────────┘   └─────────┘
                                              ↓
┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐
│ ⑧ 发布  │ ← │ ⑦ 导出  │ ← │ ⑥ 合成  │ ← │ ⑤ 配音  │
│ 分发     │   │ 渲染     │   │ 剪辑     │   │ 字幕     │
└─────────┘   └─────────┘   └─────────┘   └─────────┘
```

### 各环节行业主流方案

| 环节 | 行业工具/方案 | 关键指标 |
|---|---|---|
| ① 选题策划 | 抖音热点宝/新榜/飞瓜 | 热点响应 < 24h |
| ② 素材准备 | 片源库/录屏/下载 | 1080p+ 原片 |
| ③ AI 剪条 | 云剪辑/剪映智能剪辑 | 5-30 秒高光 |
| ④ 文案生成 | GPT/Claude + 提示词模板 | 3-5 分钟/条 |
| ⑤ 配音合成 | ElevenLabs/阿里云 TTS/讯飞 | 自然度 > 4.5/5 |
| ⑥ 字幕生成 | Whisper/剪映自动字幕 | 准确率 > 95% |
| ⑦ 剪辑合成 | FFmpeg/After Effects | 实时预览 |
| ⑧ 多平台发布 | 一键分发到抖音/B站/小红书 | 9:16/1:1/16:9 |

## 3. 优化建议 (按优先级)

### P0 — 一键成片 (One-Click Pipeline)

**现状：** 用户需手动在 5+ 个步骤间切换 (上传→分析→文案→配音→合成→导出)

**建议：** 新增 `AutoPipeline` 模式，用户只需：
1. 选择素材
2. 选择风格模板 (电影解说/第一人称/混剪)
3. 点击「一键生成」

**实现路径：**
```
新增: src/core/pipeline/steps/full-pipeline.ts
  - 串联: analyze → highlight → script → tts → compose → export
  - 进度回调: onProgress(step, percent)
  - 错误恢复: 单步重试 / 跳过
```

**预期收益：** 单条视频生产时间 30min → 5min

---

### P1 — 批量化生产 (Batch Production)

**现状：** 单视频串行处理

**建议：** 新增批量模式
- 多素材队列
- 并行处理 (最多 3 并发)
- 批量导出

**实现路径：**
```
新增: src/core/services/batch/batch-orchestrator.ts
复用: 现有 ai-clip/batch-processor.ts 模式
```

---

### P2 — 背景音乐库 (BGM Library)

**现状：** 无 BGM 功能

**建议：**
- 内置免版税 BGM 分类 (悬疑/搞笑/燃/治愈)
- 自动音量闪避 (ducking)
- AI 根据视频情绪匹配 BGM

**实现路径：**
```
新增: src/pages/workspace/components/bgm-selector.tsx
新增: src/core/services/video/bgm-service.ts
扩展: audio-mix-service.ts 增加 ducking 支持
```

---

### P3 — 模板化创作 (Template System)

**现状：** 用户每次从零配置

**建议：**
- 预设模板 (电影解说 vlog/混剪/Reaction)
- 自定义模板保存
- 模板市场 (未来)

**实现路径：**
```
新增: src/core/config/workflow-templates.ts
扩展: 现有 function-config.ts 升级为模板系统
```

---

### P4 — 平台直发 (Direct Publishing)

**现状：** 仅导出到本地

**建议：** (需平台 API 授权)
- 抖音创作者 API 直发
- B站创作平台直发
- 小红书直发

**实现路径：**
```
新增: src/core/services/publish/publish-service.ts
新增: src/pages/settings/platform-accounts.tsx
```

## 4. 立即可落地的改进 (Quick Wins)

以下改动范围小、收益高，可在 1-2 天内完成：

### 4.1 步骤进度可视化

**现状：** 用户不知道当前在流程哪一步

**改动：** 在 `edit-step/step-list.tsx` 增加状态指示
- 已完成步骤：✓ 绿色
- 当前步骤：● 高亮
- 未来步骤：○ 灰色

### 4.2 智能默认值

**现状：** 用户需手动选择分辨率/格式

**改动：** 根据素材自动推荐
- 竖屏素材 → 默认 9:16
- 横屏素材 → 默认 16:9
- 方形素材 → 默认 1:1

### 4.3 草稿自动保存

**现状：** 用户可能丢失进度

**改动：** 利用现有 `useAppStore.autoSave` + 项目持久化
- 每 30s 自动保存草稿
- 异常退出后恢复

### 4.4 预览播放器增强

**现状：** VideoPlayer 已接入但功能基础 (commit 746a80c)

**改动：**
- 时间线预览 (点击时间线跳转)
- 实时效果预览
- 音频波形显示

```
Week 2:  P1 批量化 + Quick Wins
  ├── Day 1-2: batch-orchestrator.ts
  └── Day 3-5: 步骤进度 + 智能默认值 + 草稿保存

Week 3:  P2 BGM + P3 模板
  ├── Day 1-3: BGM 库 + 音量闪避
  └── Day 4-5: 模板系统

Week 4+: P4 平台直发 (需商务对接)
```

## 6. 风险与缓解

| 风险 | 缓解 |
|---|---|
| 一键成片失败率高 | 单步重试 + 跳过机制 |
| 批量处理资源占用 | 并发限制 + 队列 |
| BGM 版权 | 仅内置免版税素材 |
| 平台 API 变动 | 抽象 publish-service 接口 |
